Testing the sum function

// We can test to see that our functions work!
function sum(numbers) {
  var sum = 0;
  for (var i = 0; i < numbers.length; i++)
    sum += numbers[i];

  return sum;
}

var numbers = [1, 5, 9, 20];
var total = sum(numbers);
if (total == 35)
  document.writeln("Sum function works!");

Note on document.writeln

  • There is a matching document.write() function. document.writeln() adds a \n (newline) character to the end of the string.
  • Newline is not interpreted by HTML because it is not white space sensitive
  • You'll notice in the exercises that I do document.write("
    ") at the beginning and document.write("
    ")
  • Within the pre tag, we are white space sensitive and so document.writeln() behaves as expected.

Function expressions

// An exponent declared function
var exponent = function(number, power) {
  // For brevity, let's ignore negative exponents
  var answer = 1;
  for (var i = 0; i < power; i++) {
    answer *= number;
  }

  return answer;
}

How is this different? The implication is that function expressions must be evaluated before the function can be called. The function is now accessible by using the variable called exponent.

Anonymous functions

In the prior example of function expressions, we were actually using an anonymous function. We just happened to assign it to a variable. What if we didn't? 

// Calling an anonymous function
(function(message) {
  alert(message);
})("Hello, World!");
}

Anonymous functions - Call backs

We just defined and executed an anonymous function. This example is a bit contrived just to demonstrate the syntax, so let's take a look at how anonymous functions might be useful to us. 

// Using anonymous functions for callbacks
function elaborateRoutine(nextSteps) {
  // Here we can do our long and elaborate task asynchronously

  // And when we're done, we invoke the next instruction
  nextSteps();
}
