Exercise 2.1.1

// Exercise 2.1.1 - Ask the user for 2 numbers. Tell them if the first or second number was bigger.
var num1 = prompt("What's the first number?");
var num2 = prompt("What's the second number?");

if (num1 > num2) {
  var largerNum = num1;
  alert(largerNum+" is larger")
}
if (num2 > num1) {
  var largerNum = num2;
  alert(largerNum+" is larger")
}
else {
  alert("I'm confused");
}

Stuff I really liked

  • Great camel case! (thisIs anExampleOf camelCase)
  • Good spacing between clauses (e.g. num1 > num2 instead of num1>num1)

Stuff to watch out for

  • You can get very different behavior with two ifs than an else if
  • Else in this situation probably means equality.

Arrays

  • Arrays have no type in JavaScript
  • Arrays are of dynamic length
  • Arrays can be added together with basic Math operations

Array syntax

// Creating an array
var list = [1, 2, "three"];
list[4] = "And now there's a fourth element";

// Getting items out of an array (they are 0 indexed)
alert(list[0]);       // Shows 1
alert(list[3]);       // Shows "And now there's a fourth element"

// Length of an array
alert(list.length);   // Alerts 4

// Adding to an array
list.push("A new item");

// Removing from an array
list.pop();

Loops

  • JavaScript uses C-style loops, plus a couple
// For loops
for (var i = 0; i < 10; i++)
  alert(i);

// For each loops
var list = ["a", "b", "c"];
for (var index in list)
  alert(list[index]);

// While loops
while (someCondition) {
  // Keep on doing this until
}

Break and continue

// Break
while (true) {
  break; // This would have been an infinite loop but broke tosses us out of it
}

// Continue
for (var i = 0; i < 10; i++) {
  if (i == 2) {
    continue; // Do nothing at the value of 2. Instead, go to the next iteration of the loop.
  }
  alert("The number is " + i);
}

Functions

  • Declared functions are a lot like functions as you'd find in C
  • A big difference is that you don't define the return type or the types of the parameters because we don't need to
  • Declared functions are defined at parse time

An example function

// An exponent declared function
function exponent(number, power) {
  // For brevity, let's ignore negative exponents
  var answer = 1;
  for (var i = 0; i < power; i++) {
    answer *= number;
  }

  return answer;
}

Note: there's already a function that does this for you. It's Math.pow(number, power). Check it out!
