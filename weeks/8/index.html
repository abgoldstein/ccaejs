function Dog(species, name, age, houseBroken) {

Prototypes

  • In JavaScript, Objects have a "parent class" called the Prototype. It is the cookie cutter from which all future Objects are made.
  • By default, the parent of an Object is just {}, an Object
  • Just like looking for variables in scope, Objects will look "up" to find a property or function that was defined previously. This is called the prototype chain
  • The prototype chain of our Dog class
    • Dog
    • Object.prototype
    • null
  • If you're adding a function to a class, best to add it via the prototype. Otherwise, it will be redefined every time for every instantiation

This is a tough concept

  • Grasping prototypical inheritance is a huge barrier to entry for many programmers
  • Once you've got this, along with scope, closures and anonymous functions (all of which we've previously visited), you've basically seen it all
  • You'll also get a chance to see this again in CodeAcademy
  • So bear with me and think through this hard. Play through any questions you have by writing code.

Inheritance

  • Prototypes and constructors are mutable
  • Meaning, we can set the prototype of one class to be the constructor of another
    • Dog.prototype = new Mammal();
    • Now Dog inherits from the Mammal class

Object utilities: typeof

// Finding the type of values
typeof 10; // Number

Object utilities: hasOwnProperty

// Checking if an object has a given field
var dog = {
  species: "dog",
  name: "Murphy",
  age: 14,
  houseBroken: true
};

dog.hasOwnProperty("favoriteGame") // false
dog.favoriteGame // undefined
