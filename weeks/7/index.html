Multidimensional arrays

  • Exactly like every other language. We're nesting arrays inside other arrays.
  • They don't come up extremely frequently (in my experience). Often there's more a combination of nested Objects within Arrays.
// 1-dimensional list of all names in the class
var names = ["Doug", "David", "Cassandra", "Shelby", "Emily", "Rada", "Sara", "Gretchen"];

// 2-dimensional list of names in the class by row where people sit.
var namesByRow = [
  ["Doug", "David"],
  ["Cassandra", "Shelby", "Emily"],
  ["Rada", "Sara", "Gretchen"]
];

Objects

  • If you are familiar with OOP, objects will feel comfortable.
  • JavaScript objects are prototypical. We'll visit that next week.
  • The short version: Objects are a collection of related data and functions.
// An example of a car object
var car = {
  make: "Honda",
  model: "Accord",
  year: 2003,
  drive: function() {
    // Vroom.
  }
};

Creation and access

  • There are two ways to create Objects:
    • Object literal notation, var object = {};
    • Object constructor, var object = new Object();
    • In my experience, object literal notation is more common and preferred.
  • There are two ways to access or modify members of the object:
    • Dot notation, object.member
    • Square bracket notation, object[member]
    • Dot notation is syntactic sugar for square bracket notation. At its heart, Objects are syntactic sugar of Arrays. Remember that comment early on about "associative arrays"?
    • In my experience, dot notation is more common and preferred.

A bit more code

// An example of a car object
var car = {
  make: "Honda",
  model: "Accord",
  year: 2003,
  brag: function() {
    alert("Hello, I am a " + this.make + " " + this.model);
  },
  pullOver: function() {
    if (this.registration) {
      alert("Here you are officer..." + this.registration);
    } else {
      // uh oh
    }
  }
};

// Let's use the car
car.drive();
car.pullOver(); // What happens?
